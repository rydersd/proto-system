Sprint 14 Board
Summary
A Kanban-style project board showing tasks across four workflow stages: Backlog, In Progress, In Review, and Done. Each card displays task title, category badge, priority level, assignee avatar, and optional metadata like progress bars, blocked status, and comment counts. Designed to demonstrate the proto-framework's card, badge, and paper utility components in a horizontally-scrolling lane layout.
JTBD
An engineering team lead needs to visualize sprint progress at a glance — seeing which tasks are blocked, who is working on what, and how work is distributed across stages to identify bottlenecks and rebalance assignments.
Design Spec
Full-width horizontal scroll layout with four flex columns (280-320px each). Lane headers use ds-card with left accent borders color-coded by stage. Task cards use ds-card with inline badge groups and avatar circles. Paper utilities (wf-tape, wf-pin, wf-torn-bottom, wf-sketch) add wireframe texture across cards. Confidence attributes mark validated vs speculative designs. Done column cards use reduced opacity and strikethrough titles. Filter pills at top use btn-ghost with pill border-radius. Progress bar on autocomplete card uses token-based accent fill on tint background.

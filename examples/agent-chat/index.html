Agent Chat Interface
Summary
A split-panel agentic AI chat interface with a conversation sidebar and a rich message area. Demonstrates structured agent responses with embedded data tables, action buttons, streaming states, and agent status indicators.
JTBD
A sales professional needs to quickly analyze pipeline data, get AI-driven recommendations, and take action — all within a conversational interface backed by specialized agents.
Design Spec
Layout: 260px sidebar + flexible main chat area (CSS grid). Sidebar shows conversation history and available agents with status badges. Main area has a fixed header, scrollable message thread, and a pinned composer at the bottom. Agent responses use structured content: text, tables (wf-table), badges (wf-badge-*), and inline action buttons (btn-primary, btn-ghost). Streaming/thinking states use dashed borders and a three-dot typing indicator. The data-wf-confidence attribute differentiates confirmed responses from uncertain/in-progress ones. Conversation list uses tape decorators (wf-tape) on the active item. Agent status card uses sketch treatment (wf-sketch). The in-progress status message uses pin decorator (wf-pin). Composer includes a contenteditable input with tool buttons (attach, mention, slash command) and a primary send button.

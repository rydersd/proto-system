Framework Wars: CSS vs. The World

By A. Designer · March 8, 2026 · 5 min read

In a stunning turn of events, the design systems community has declared a new era of token-driven development. Sources close to the specification committee confirm that hardcoded hex values are now considered a Level-3 violation, punishable by mandatory wireframe review sessions.

“We always knew this day would come,” said a senior prototype architect, speaking on condition of anonymity. “The tokens were there all along — we just needed the courage to use them.”

The Rise of Design Tokens

The movement began quietly in 2023, when a small group of front-end engineers started replacing inline hex colors with CSS custom properties. What began as a maintenance convenience quickly evolved into a philosophical position: design intent should be expressed through semantic names, not arbitrary values.

“Hardcoding #3d6daa is like naming a variable x. Design tokens are self-documenting.” — Dr. S. Token, Design Systems Institute

The implications rippled across the industry. Component libraries that once shipped with baked-in colors were retrofitted with token layers. Theming systems that required complete CSS overrides now changed behavior with a single JSON file swap. The era of “just override the hex” was over.

The Fidelity Question

Perhaps the most contentious aspect of the framework wars has been the question of fidelity. Traditional tools push designers toward pixel-perfect mockups from day one. The wireframe movement argues that this creates false confidence — stakeholders mistake polish for progress.

The nib framework introduced a fidelity slider that lets prototypes exist in three states: napkin (rough, sketchy), blueprint (structured, precise), and polished (refined, near-production). Each state serves a different conversation — from early ideation to final review.

“The napkin mode changed how we present to executives,” said one product manager. “They stopped debating button colors and started debating user flows. That’s the whole point.”

What’s Next

The specification committee is expected to release formal guidance by Q3, including recommended token naming conventions, a reference implementation in CSS custom properties, and tooling for automated compliance checking. Several major design tool vendors have already committed to native support.

Until then, the wireframe community continues its work: building, testing, and iterating — one token at a time.

# Research Study — Nib Example

A starting scaffold for research-mode prototypes: card sorts, persona libraries, JTBD maps, interview synthesis. Lifted from `eqPartners/prototype/card-sort.*` and de-branded; pairs with the reference Cloudflare Worker's `POST /api/card-sort` endpoint.

## What ships

- **`card-sort.html`** — closed card-sort runner. Loads `data/card-sort-<study>.json`, presents cards in a tray + buckets, drag-to-place, submit posts to the configured endpoint.
- **`data/card-sort-taxonomy.json`** — example study (12 cards, 5 buckets). Replace with your own.
- **`index.html`** — landing page; lists available studies + stub callouts for personas and JTBD pages.

## Adding a new card-sort study

1. Drop a JSON file at `data/card-sort-<your-study-id>.json`:

```json
{
  "study_id": "your-study-id",
  "title": "Study title",
  "description": "One-sentence prompt for participants.",
  "cards":   [{ "id": "c1", "label": "Card text" }],
  "buckets": [{ "id": "b1", "label": "Bucket label" }]
}
```

2. Open `card-sort.html?study=your-study-id` in a browser served via HTTP.

## Recording results

Submissions POST to `/api/card-sort` on the same origin. The reference worker (`examples/cloudflare-worker/`) handles this endpoint and stores each submission as a GitHub issue labeled `card-sort:<study-id>`.

To analyze results, query the GitHub Issues API filtered by `label:card-sort:<study-id>`. Each issue body is a JSON dump of the participant's placements.

## Next: personas + JTBD pages

The two stub callouts on `index.html` are placeholders for:

- A persona library page reading `data/personas.js` (generated by `nib-ingest` from the workbook's `personas` tab).

Both are straightforward static pages — see [`examples/spreadsheet-bootstrap/index.html`](../spreadsheet-bootstrap/index.html) for a worked example of consuming the generated data files.

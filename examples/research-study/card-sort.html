Summary

Closed card-sort study runner, built on React Flow. Default study is the partner-taxonomy sort defined in data/card-sort-taxonomy.json; participants drag cards from the tray on the left into one of four buckets on the right. On submit, all placements POST to /api/card-sort on the Cloudflare Worker.

  • Reuse the same React Flow + card pattern for any future closed-sort study by adding another card-sort-<name>.json file and linking with ?study=<name>.

Design Spec

  • Tray on the left (unsorted cards stack vertically), buckets on the right (each grows as you drop cards in).
  • Drag a card → snap to whichever bucket its center is in on drag-stop. Drop outside any bucket → it returns to the tray.
  • Progress count in the topbar: "X / Y sorted." Submit button enabled once at least 1 card is placed; label warns if any remain in the tray.
  • On submit, success state replaces the canvas with a thank-you message.
  • Open card-sort variant (let participants create their own buckets) is a future iteration.

Technical Details

  • React 18 + ReactFlow 12 from esm.sh, no build step.
  • Study definition fetched from data/card-sort-<id>.json — defaults to taxonomy, override with ?study=foo.
  • Submission payload: {study_id, started_at, finished_at, placements: {card_id: bucket_id|null}, user_agent}.
  • Worker route POST /api/card-sort stores each submission as a GitHub issue with label card-sort:<study_id> for downstream aggregation.
  • Aggregation page (compute the >70% threshold per card) is on the backlog.

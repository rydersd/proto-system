Summary

The Design Stories page is a living document that tracks implementation scope, product decisions, phased delivery, and platform approach for each design story. It auto-generates from DESIGN_STORIES defined in project-data.js.

Design Specification

Primary Content

Auto-validated story cards with user stories, acceptance criteria, phased implementation tables, decision logs, and SFDC implementation suggestions. Project roadmap shows how stories group into delivery phases.

Interactive Elements

  • Status filter pills — filter stories by draft/in-progress/accepted/deferred
  • Decision log accordions — expand/collapse per story
  • Page links — navigate to wireframe pages where stories are realized
  • Journey links — navigate to user-flows.html for related journeys
  • View-control nav — links to sibling reference pages

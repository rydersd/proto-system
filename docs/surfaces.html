Surfaces

Platform-specific CSS overlays that adapt the wireframe to different contexts.

Rule: Use one surface CSS per page. Don't mix Slack + Salesforce on the same page.

What Are Surfaces?

Surfaces are CSS overlays that add platform-specific styling on top of the core wireframe components. They provide the look and feel of specific platforms (Slack, Salesforce, internal portals) while maintaining the wireframe aesthetic.

Each surface provides its own set of prefixed components that complement the base wf- components:

Internal Design System (internal-ds.css)

For internal portals, dashboards, and admin tools. Uses the ds- prefix.

<link rel="stylesheet" href="surfaces/internal-ds.css">

Key Components

  • .ds-card — Card with header/body structure
  • .ds-kpi-card — KPI metric card with label/value/delta
  • .ds-kpi-grid — Auto-fit grid for KPI cards
  • .ds-sidebar-card — Sidebar info card
  • .ds-card-title — Card header title

Slack (slack.css)

For Slack app wireframes — messages, threads, channels, app home. Uses the slack- prefix.

<link rel="stylesheet" href="surfaces/slack.css">

Key Components

  • .slack-shell — Full Slack app container with sidebar + main
  • .slack-sidebar — Channel navigation sidebar
  • .slack-message — Chat message with avatar + content
  • .slack-thread — Thread container
  • .slack-btn-primary — Slack-styled primary button
  • .slack-composer — Message input area

Salesforce (salesforce.css)

For SFDC record pages, Lightning-style layouts. Uses the sfdc- prefix.

<link rel="stylesheet" href="surfaces/salesforce.css">

Key Components

  • .sfdc-record-page — Full record page layout
  • .sfdc-record-header — Record header with title + actions
  • .sfdc-path-bar — Sales path progress bar
  • .sfdc-highlights-bar — Key field highlights
  • .sfdc-card — Lightning card
  • .sfdc-btn-primary — SFDC-styled button
  • .sfdc-feed-item — Chatter feed item

Choosing a Surface

When creating a wireframe, choose the surface that matches the target platform:

Building ForSurfaceUse When
Internal tools internal-ds.css Dashboards, admin panels, portals
Slack apps slack.css Bot messages, app home tabs, modals
Salesforce salesforce.css Record pages, Lightning components

Creating a New Surface

To add a surface for a new platform:

  1. Create surfaces/your-platform.css
  2. Use a unique prefix for all classes (e.g., .jira-, .notion-)
  3. Build on top of core tokens — never hardcode hex values
  4. Add wireframe-mode overrides: html.wireframe .your-component { ... }
  5. Add napkin-mode button overrides for sharpie aesthetic
  6. Create a reference doc at ref/surface-your-platform.md
  7. Create a starter at starters/your-platform-starter.html

Getting Started

Bootstrap a new wireframe project in 5 minutes.

Prerequisite: You need a copy of the proto-system framework. The core/ and surfaces/ directories must be accessible from your project.

1. Copy a Starter

The starters/ directory contains ready-to-use HTML templates for different surface types. Copy one to your project directory:

cp starters/internal-ds-starter.html my-project/dashboard.html

Available starters: internal-ds-starter.html, slack-starter.html, salesforce-starter.html

2. Create project-data.js

Every project needs a project-data.js file that defines the page structure. This drives the navigation drawer automatically.

/* project-data.js */
window.WIREFRAME_CONFIG = {
  title: 'My Project',
  subtitle: 'Wireframe Prototype'
};

window.SECTIONS = [
  {
    epic: 'E1',
    label: 'Core Screens',
    items: [
      { file: 'dashboard', label: 'Dashboard' },
      { file: 'settings',  label: 'Settings' }
    ]
  }
];

3. Script Load Order

Scripts must load in this order at the bottom of your HTML:

<script src="project-data.js"></script>
<script src="../core/proto-nav.js"></script>

project-data.js sets window.SECTIONS and window.WIREFRAME_CONFIG. proto-nav.js reads these to build the context bar, navigation drawer, and design notes panel.

4. Design Tokens

Never hardcode hex colors. Always use CSS custom properties (tokens):

/* DO — uses token */
color: var(--wf-ink);
background: var(--wf-surface);

/* DON'T — hardcoded hex */
color: #1e2a3a;
background: #edf1f7;

Tokens automatically adapt across fidelity modes (napkin, blueprint, polished). See Design Tokens for the full palette.

5. Paper Utilities

Add physical-artifact texture with CSS classes:

  • .wf-tape — tape strip across top center
  • .wf-pin — pushpin dot at top center
  • .wf-torn-top / .wf-torn-bottom — torn paper edge
  • .wf-stacked — stacked paper pile effect
  • .wf-sketch — heavy hand-drawn border
<div class="ds-card wf-tape">
  <!-- Card with tape decoration -->
</div>

6. Fidelity Slider

The context bar includes a fidelity slider with three modes:

  • Napkin — rough, hand-drawn look. White paper + sharpie aesthetic. Grayscale.
  • Blueprint — default. Structured but clearly a wireframe.
  • Polished — clean, minimal wireframe artifacts.

The slider state persists in sessionStorage across page navigations within the same session.
